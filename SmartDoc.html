🎯 Objectif

Transformer un dossier documentaire non structuré en mémoire sourcée, contextualisée et maintenue par une routine de classement incrémental.

Règle centrale : Claude ne supprime jamais un fichier et ne déplace rien sans validation humaine explicite.

1. 🧱 Créer le projet

Nom proposé : Proto Claude SmartDoc

Prompt — Instructions projet

Ce projet sert à prototyper la gestion documentaire d'un fonds hétérogène (conférences, articles, notes, correspondances, documents institutionnels).

Règles générales :
- ne jamais supprimer un fichier ;
- ne jamais modifier le contenu d'un document source ;
- ne jamais renommer ou déplacer un document sans validation explicite du lot par l'archiviste ;
- ne jamais inventer de métadonnée manquante ; utiliser unknown, laisser vide ou demander validation ;
- enregistrer une date uniquement si elle est explicitement présente dans le document ou le nom de fichier, en indiquant son origine ;
- ne jamais inscrire une date déduite par rapprochement sans validation explicite de l'archiviste ;
- travailler par étapes simples ;
- tenir un registre Excel comme mémoire documentaire ;
- utiliser des colonnes et valeurs en anglais dans le fichier Excel ;
- traiter uniquement les nouveaux éléments ou les éléments modifiés ;
- limiter les volumes traités à chaque exécution ;
- produire un rapport court à chaque run ;
- utiliser la skill xlsx-proto pour créer, lire et modifier le fichier Excel ;
- importer et vérifier la skill vizualise-documentation à l'étape 2 ;
- ne créer la Skill métier documentation qu'après plusieurs étapes testées dans ce fil ;
- créer la route finale seulement après validation du workflow et de la Skill métier.

10. ❓ Produire des questions de recherche

Objectif : réactiver la mémoire Excel validée à partir d'un ajout ou d'une date inconnue pour produire des pistes de recherche.

Prompt — Questions de recherche contextualisées

À partir du registre documentaire validé, prends un document sans date certaine (date_status = unknown ou to_investigate) ou un nouveau document ajouté.

Réactive la mémoire Excel en comparant :
- les thèmes et concepts du document ;
- le vocabulaire utilisé ;
- les personnes citées ;
- l'auditoire visé ;
- le destinataire ;
- le genre documentaire.

Génère 2 à 5 questions de recherche pour l'archiviste, par exemple :
- "Ce document destiné à X et mobilisant les concepts Y/Z peut-il être rapproché du cycle de conférences de telle période ?"
- "L'évolution du concept A entre les documents B et C éclaire-t-elle la date de ce document ?"
- "Le vocabulaire de ce document est-il cohérent avec la période des documents D et E ?"

Pour chaque question, indique :
- sa formulation ;
- les sources activées (documents de référence du registre) ;
- sa justification ;
- son niveau de confiance (high / medium / low) ;
- la limite d'interprétation.

Règles :
- ne jamais inscrire une date ou période proposée dans le registre ;
- seule une réponse ou une validation explicite de l'archiviste peut modifier detected_date et date_status ;
- les questions ne valent jamais conclusion ;
- présente les questions comme des pistes de recherche, pas comme des affirmations.
